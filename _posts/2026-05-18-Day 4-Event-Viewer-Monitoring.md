### Wireshark Capture

Wireshark captured SMB2 KeepAlive requests and responses between the two VMs.  
This showed that the SMB session remained active after authentication and continuously exchanged network traffic.

![image](/assets/wiresharksmb.png)


### Why this matters for Defender

Failed SMB authentication attempts are commonly monitored in enterprise environments because they may indicate brute-force attacks or unauthorized access attempts.


### Reflection

This lab helped me better understand how Windows records authentication activity and how different tools provide different visibility during investigations.

Event Viewer showed authentication-related events, while Wireshark provided packet-level network visibility.  

I also learned that SMB-specific logs contain more detailed information than general Windows Security logs.



### Questions that came up during the lab

#### Q. Why use Event Viewer instead of a SIEM?

At first, I thought SIEM tools directly created security logs and provided all monitoring data themselves.

However, during the lab, I learned that SIEM platforms mainly collect, centralize, and analyze logs that are already generated by systems such as Windows Event Viewer, Sysmon, firewalls, or applications.


Windows / Sysmon / Applications
        ↓
     Generate Logs
        ↓
   Forward to SIEM
        ↓
 Search, Correlation, Alerts
